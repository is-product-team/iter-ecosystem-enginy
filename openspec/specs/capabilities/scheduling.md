# Specification: Workshops & Scheduling

This capability manages the entire lifecycle of educational workshops, from center requests to session execution and student enrollment.

## 1. Lifecycle of a Workshop Assignment

An assignment transitions through several states, each requiring specific actions:

1. **`PENDING` / `PROVISIONAL`**: Initial request from a center or auto-generated by the AI motor.
2. **`PUBLISHED`**: Confirmed by an administrator. Centers can now see it.
3. **`DATA_ENTRY`**: Centers designate two **Referent Teachers** and upload the **Nominal Register** (student list).
4. **`VALIDATED`**: Admin verifies the documentation (manually or via Vision AI).
5. **`IN_PROGRESS`**: After legal registration is confirmed, sessions are automatically generated and the workshop begins.
6. **`COMPLETED`**: All sessions finished and evaluations submitted.

## 2. Fair Distribution Algorithm (AI Motor)

To handle cases where demand exceeds workshop capacity (especially in **Modalitat C**), the `AutoAssignmentService` implements a fairness logic:

- **Base Share**: Capacity is split evenly among all requesting centers.
- **Priority Remainder**: Leftover spots are distributed based on a "First-Come, First-Served" (FCFS) priority using the request timestamp.
- **Round Robin**: In case of odd lots, a round-robin distribution ensures no center is unfairly excluded.

## 3. Vision AI Document Validation

The `VisionService` provides automated analysis of uploaded documents (e.g., Pedagogical Agreements):
- **Format Check**: Ensures files are valid PDFs.
- **Signature Detection**: Uses simulated object detection to verify signatures in the signature box region.
- **Confidence Scoring**: Returns a confidence level (e.g., 0.98 if signed) to support or automate administrative approval.

## 4. Automatic Session Generation

Once an assignment is confirmed for start (`IN_PROGRESS`), the system generates individual sessions based on the workshop's `dies_execucio` (execution days):
- **Scheduling**: Maps the weekly slots to actual calendar dates within the active program phase.
- **Staffing**: Automatically assigns the referent teachers to the generated sessions.

## 5. Operational Checklist

Each assignment has a mandatory checklist to ensure compliance:
- [ ] Designate Referent Teachers.
- [ ] Upload Nominal Register (Excel).
- [ ] Sign Pedagogical Agreement (Required for Modality C).

Administrators can track these across all centers to identify bottlenecks.
