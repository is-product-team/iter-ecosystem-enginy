## ADDED Requirements

### Requirement: ESM Module Resolution in Shared Package
The `shared` package SHALL use explicit `.js` extensions for all internal relative imports to ensure compatibility with Node.js ESM mode in production.

#### Scenario: Successful module resolution
- **WHEN** the API starts in production mode
- **THEN** it resolves the `shared` package imports without `ERR_MODULE_NOT_FOUND` errors

### Requirement: Unified Production Build Structure
The Docker build process SHALL copy the compiled `dist` directory in a way that preserves the relative path structure between `apps/api` and `shared` as generated by the TypeScript compiler.

#### Scenario: Entry point availability
- **WHEN** the Docker container starts
- **THEN** the entry point at `apps/api/dist/apps/api/src/index.js` is found and executed correctly

### Requirement: Resilient Environment Loading
The API SHALL prioritize system environment variables over physical `.env` files and SHALL NOT crash if a `.env` file is missing, provided that all required variables are present in the environment.

#### Scenario: Startup without .env file
- **WHEN** the API starts in an environment where all required variables are set but no `.env` file exists
- **THEN** the API starts successfully and passes the health check

### Requirement: Infrastructure Completeness
The production infrastructure SHALL include an `ollama` service reachable by the API for document validation tasks.

#### Scenario: AI Service Connectivity
- **WHEN** the API attempts to connect to the `ollama` service in production
- **THEN** the connection is established or handled gracefully with a standard fallback
